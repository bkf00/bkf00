## Engineering style

I like systems that are:

- **boring in production** - predictable, observable, and easy to reason about;
- **clear in code review** - names, boundaries, and tests should explain the intention;
- **practical before fancy** - the best abstraction is the one that pays rent;
- **built with users in mind** - even internal tools deserve good flow and low friction.
